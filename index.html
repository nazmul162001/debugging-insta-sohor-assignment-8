JavaScript is a heigh level interpreted and single threaded programming language. which means it can handle one task at a time. When we write the javascript code firstly it goes to the browser. Then js engine converts it into a machine language to make understood for machine. After that line by line machine code is executed and eventually it gives the output to our computer.

Javascript executes codes one peach of task at a time in the call Stack.But when an asynchronous task like as web api task is on call stack,it cannot be executed immediately and it is delivered to Web API Stack and further codes of call stack are executed. When it is time to execute to code of web API, it is headed to call back queue then delivered to call stack to execute further when it is time and executed and eventually asynchronous task execution is finished. In summery we can say that Event loop is responsible for checking the call stack in the event queue,also called as callback queue, if all the statements from the call stackhave finished execution event loop will pop the statementfrom even queue and push it to the call stack.

Local Storage
  • All Data is persisted when the browser session ends
  • It's not sessions based, need to be deleted via JS or manually
  • It allows storing data with no expiration date
  • Designed for storage that spans multiple windows and it lasts beyond the current session
Session Storage
  • All Data is cleared when the browser session ends
  • It is session based and working per window or tab
  • It allows storing data for one session
  • It Allow the user to carry out a single transaction but could be carrying out multiple transactions in different windows at the same time
