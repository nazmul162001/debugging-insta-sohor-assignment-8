Frequently Asked Question (FAQ)

JavaScript is a single threaded and single concurrent programming language which means it can handle one task at a time or, in other words, a piece of code at a time. It's an heigh level interpreted programming language, and like most scripting languages, it uses dynamic typing, where type safety is verified at the runtime.The source code is passed through a program called a compiler, which translates it into bytecode that the machine understands and can execute.

Event loop is responsible for checking the call stack in the event queue also called as callback queue, if all the statements from the call stack have finished execution event loop will pop the statement from the event queue and push it to the call stack.

Local Storage
  • Data is persisted when the browser session ends
  • It's not sessions based, need to be deleted via JS or manually
  • It allows storing data with no expiration date
  • Designed for storage that spans multiple windows and it lasts beyond the current session
Session Storage
  • Data is cleared when the browser session ends
  • It's session based and working per window or tab
  • It allows storing data for one session
  • Allows the user to carry out a single transaction but could be carrying out multiple transactions in different windows at the same time
